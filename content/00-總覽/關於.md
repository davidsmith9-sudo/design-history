<span class="eye-section-label">Why · 為什麼</span>

## 為什麼存在這個站

中文設計史的線上資源**長期分散**:

- 維基百科條目結構不一、缺乏交叉索引
- 學術期刊難取得、英文資料中譯品質參差
- 設計教科書翻譯落後 5–10 年
- 圖文整理散落在臉書社團、設計師個人部落格

我希望這個站能填補的空白是:

1. **繁體中文一手書寫** — 不是機器翻譯,有寫作者親自下筆的觀點
2. **跨類別交叉索引** — 從一個流派一鍵跳到代表設計師,再跳到代表作品
3. **編輯立場明確** — 不假裝中性,有「為什麼這個流派比那個重要」的判斷
4. **狀態誠實透明** — 草稿就標草稿,不裝完整(見 [Status 徽章說明](/00-總覽/授權))

<span class="eye-section-label">How · 方法論</span>

## 怎麼選的:21 流派 / 48 人物 / 36 作品

### 流派篩選標準
- **持續至少 5 年**且有**集體創作宣言**或**清晰風格主張**
- 對**後續設計實踐**有可追溯影響(出現在多本主流設計史教科書索引)
- **跨域**:不限於建築 / 平面 / 工業 / 字體之一
- 排除:單一設計師個人風格(歸入「人物」)、商業 trend(歸入「時代背景」)

### 人物篩選標準
- 至少有**一件被多本教科書收錄**的代表作
- 或**理論貢獻**(著作、宣言、教育影響)被後輩持續引用
- 含**教育者 + 評論家**,不限「實踐型設計師」
- 嘗試**地理多樣性**(歐 / 美 / 東亞 / 拉丁美洲各有代表),但仍偏歐美中心 ── 見下方 Limitations

### 作品篩選標準
- 多本設計史教科書同時收錄,或
- **單一作品定義一個流派**(如薩伏伊別墅之於國際樣式)
- **跨領域 anchor**:能串接到其他人物、流派、理論
- 排除:純商業成功(銷量、知名度)但無設計史意義者

<span class="eye-section-label">Editorial · 編輯決策</span>

## 為什麼這樣設計

整站的視覺語言 — Norm / Frama 風的編輯型極簡 — 是有意識的選擇:

- **紙白 + 石墨 + 髮絲線**:致敬北歐戰後設計與瑞士國際主義的純粹性
- **Oxblood `#7A2E2A` 為唯一強調色**:呼應印刷編輯傳統(章節標、引用標、強調區)
- **Fraunces serif + Noto Serif TC**:英文活潑、中文穩重,兼顧雙語節奏
- **YearRail 常駐年代軸**:本站獨有設計記憶點 — 提醒讀者「你看的是 774 年大圖中的哪一格」
- **交叉索引 wikilink graph**:從 Obsidian 工作流借來的概念,讓「**從一個 → 跳到另一個**」是核心動作

完整 case study 規劃中。

<span class="eye-section-label">Limitations · 局限</span>

## 我做不到的事(誠實聲明)

這份筆記**故意 / 不得不**留下的空白:

- **東南亞、非洲、印度設計**內容稀薄 ── 因為我接觸的二手資料偏歐美
- **女性設計師**比例偏低 ── 反映設計史教科書本身的結構性偏差,我有意識但目前還沒完全平衡
- **商業設計、廣告、UX 早期史**幾乎沒有 ── 不是不重要,是我背景偏建築 / 工業 / 平面
- **20 世紀後半的東亞設計**(台、日、韓、港、中)目前散在「人物」和「作品」,還沒有系統性整理
- **數位 / 介面 / 服務設計**只有點到 iPhone 為止 ── 1990s 後我覺得時間還不夠長,設計史評斷尚未沉澱
- 我**寫得快不過讀得快** ── 翻幾本書、看一個演講、看幾天展可能改寫一篇條目。所以你看到的不是「定本」,是「**正在學的人寫給正在學的人**」的筆記

> **看到我做錯 / 做偏 / 該補的,請[直接提 Issue](https://github.com/davidsmith9-sudo/design-history/issues)**。這個站希望被質疑、被指正,不是被默默使用而後遺忘。
